# Pattern: Accessibility

**Mandatory for every View file.** App Store review rejects apps with critical accessibility issues.
Violations are treated the same as build errors — fix before Phase 4.

---

## Core Rules (A-1 to A-6)

### A-1 — Every interactive element must have an accessibilityLabel

```swift
// ❌ VoiceOver reads "Button" — useless
Button { viewModel.submit() } label: {
    Image(systemName: "arrow.right")
}

// ✅ VoiceOver reads "Submit form"
Button { viewModel.submit() } label: {
    Image(systemName: "arrow.right")
}
.accessibilityLabel(L10n.A11y.submitButton)
```

### A-2 — Every UITest-targetable element must have an accessibilityIdentifier

```swift
// ✅ Required for visual verification (Phase 3.6) and UITests
TextField(L10n.Placeholder.email, text: $viewModel.email)
    .accessibilityIdentifier("email_input")

Button(L10n.Action.signIn) { Task { await viewModel.login() } }
    .accessibilityIdentifier("sign_in_button")

ProgressView()
    .accessibilityIdentifier("loading_indicator")
    .accessibilityLabel(L10n.A11y.loading)
```

**Naming convention for identifiers:** `snake_case`, descriptive, screen-scoped if needed.
- `login_email_input`, `login_password_input`, `login_sign_in_button`
- `home_product_list`, `home_user_greeting`

### A-3 — Decorative images must be hidden from VoiceOver

```swift
// ✅ VoiceOver skips purely decorative images
Image("HeroBackground")
    .accessibilityHidden(true)

// ✅ SF Symbol with semantic meaning
Image(systemName: "checkmark.circle.fill")
    .accessibilityLabel(L10n.A11y.successIcon)
```

### A-4 — Loading states must be announced

```swift
// ✅ VoiceOver announces when loading starts and stops
ProgressView()
    .accessibilityLabel(L10n.A11y.loading)

// ✅ For in-place loading (e.g. refreshing a list)
List { … }
    .accessibilityValue(viewModel.isLoading ? L10n.A11y.loading : "")
```

### A-5 — Error messages must be readable by VoiceOver

```swift
// ✅ Error surfaced to VoiceOver
if let message = viewModel.errorMessage {
    Text(message)
        .foregroundStyle(.appError)
        .accessibilityLabel(message)          // explicit even if same as text
        .accessibilityAddTraits(.isStaticText)
}
```

### A-6 — Dynamic Type must not break layouts

```swift
// ❌ Fixed-height containers break with large text
VStack { … }.frame(height: 44)

// ✅ Let content define height, use minimum where needed
VStack { … }.frame(minHeight: 44)

// ✅ Use scaledMetric for spacing that should grow with type
@ScaledMetric var iconSize: CGFloat = 24
```

## Grouping Elements

Group related elements so VoiceOver reads them as one unit:

```swift
// ✅ Label + value grouped as one VoiceOver element
HStack {
    Text(L10n.Label.total)
    Spacer()
    Text(viewModel.formattedTotal)
}
.accessibilityElement(children: .combine)
.accessibilityLabel("\(L10n.Label.total): \(viewModel.formattedTotal)")
```

---

## Focus Management

After a screen transition or modal dismissal, set focus to the appropriate element:

```swift
// ✅ Move focus to first field after login screen appears
@AccessibilityFocusState private var isEmailFocused: Bool

TextField(L10n.Placeholder.email, text: $viewModel.email)
    .accessibilityFocused($isEmailFocused)
    .onAppear { isEmailFocused = true }
```

## Reviewer Checklist (Phase 5)

Every View file reviewed in Phase 5 must pass:

- [ ] **A-1** All `Button`, `Toggle`, `TextField`, `Picker`, `Slider`, `Stepper` have `.accessibilityLabel`
- [ ] **A-2** All interactive elements have `.accessibilityIdentifier` (snake_case)
- [ ] **A-3** Decorative images have `.accessibilityHidden(true)`
- [ ] **A-4** `ProgressView` has `.accessibilityLabel(L10n.A11y.loading)`
- [ ] **A-5** Error message `Text` has explicit `.accessibilityLabel`
- [ ] **A-6** No fixed-height containers wrapping text — use `minHeight` or no height constraint
- [ ] Identifiers match the Accessibility Identifier Map in `01-spec.md`
- [ ] No raw English strings in `.accessibilityLabel(…)` — must use `L10n.A11y`
